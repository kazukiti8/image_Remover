# 画像クリーンアップシステム - ファイル構造

このドキュメントでは、画像クリーンアップシステムのファイル構造と各ファイルの役割について説明します。

## 概要

このシステムはモジュール構造で設計されており、以下の主要なコンポーネントで構成されています：

1. 基本クリーンアップエンジン
2. グラフィカルユーザーインターフェース（GUI）
3. 拡張機能（EXIF表示、AI評価など）
4. ユーティリティ（設定管理、ドラッグ&ドロップなど）

## ファイル一覧と役割

### コアファイル

#### `main_application.py`
メインアプリケーションを実行するための起点となるファイルです。このファイルから全ての機能にアクセスできます。

主な機能：
- アプリケーションの初期化と実行
- メニュー、ツールバー、ステータスバーの設定
- モジュール間の連携

#### `image_cleanup_system.py`
画像のクリーンアップに関する基本機能を提供するコアモジュールです。

主な機能：
- ブレている画像の検出
- 類似画像の検出
- 重複画像の検出
- 画像の処理と操作

### GUI関連ファイル

#### `thumbnail_view.py`
サムネイルグリッド表示を提供するモジュールです。

主な機能：
- 画像のサムネイル表示
- グリッドレイアウトでの表示
- サイズ変更や並び替え機能

#### `image_zoom_widget.py`
画像拡大/縮小と表示に関するウィジェットを提供するモジュールです。

主な機能：
- ズーム可能な画像表示
- マウスホイールでのズーム操作
- 画像の並べて比較機能

### 拡張機能ファイル

#### `exif_display.py`
画像のEXIF情報を表示するためのモジュールです。

主な機能：
- EXIF情報の読み込みと解析
- 情報の整理と表示
- データのエクスポート

#### `ai_quality_assessment.py`
AIを活用した画像品質評価機能を提供するモジュールです。

主な機能：
- シャープネス評価
- 露出評価
- コントラスト評価
- ノイズ評価
- 構図評価

#### `batch_processor.py`
複数のディレクトリを一括処理するためのモジュールです。

主な機能：
- 複数ディレクトリの処理
- バッチジョブの管理
- 結果のレポート作成

### ユーティリティファイル

#### `drag_drop_support.py`
ドラッグ&ドロップ操作をサポートするユーティリティモジュールです。

主な機能：
- ドラッグ&ドロップ領域の提供
- ファイルとディレクトリのドロップ処理
- 既存ウィジェットへのドラッグ&ドロップ機能追加

#### `settings_manager.py`
アプリケーションの設定を管理するモジュールです。

主な機能：
- 設定の読み込みと保存
- 設定ダイアログの提供
- 設定のインポート/エクスポート

## モジュール間の関係

```
main_application.py
    ├── image_cleanup_system.py
    ├── thumbnail_view.py
    ├── exif_display.py
    ├── ai_quality_assessment.py
    ├── batch_processor.py
    ├── drag_drop_support.py
    ├── image_zoom_widget.py
    └── settings_manager.py
```

- `main_application.py`はすべてのモジュールを統合し、全体のUIを構築します
- 各機能モジュールは独立して動作しますが、`main_application.py`を通じて連携します
- `settings_manager.py`は全モジュールの設定を一元管理します

## コード規約

このプロジェクトでは以下のコード規約を採用しています：

- クラス名: CamelCase (例: `ImageCleanupSystem`)
- メソッド/関数名: snake_case (例: `detect_blurry_images`)
- 変数名: snake_case (例: `blur_threshold`)
- 定数: 大文字のSNAKE_CASE (例: `MAX_IMAGE_SIZE`)
- インデント: 4スペース

## モジュール拡張方法

新しい機能を追加する場合は、以下の手順に従ってください：

1. 新しい機能用のPythonファイルを作成
2. 機能に必要なクラスとメソッドを実装
3. `main_application.py`にインポート文を追加
4. メインアプリケーションクラスに機能を統合

詳細な拡張方法については `EXTENDING.md` を参照してください。
